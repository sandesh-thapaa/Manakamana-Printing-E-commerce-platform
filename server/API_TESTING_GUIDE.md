# API Testing Guide for Postman

This guide contains all endpoints, required headers, and request bodies to test the completely integrated backend APIs (Client Onboarding, Auth, Templates, and Design Approval workflows). 

> **Base URL:** `http://localhost:8005/api/v1` (Adjust the port if your `.env` specifies a different one)

---

## 1. PUBLIC CLIENT APIs (No Token Required)

### 1.1 Submit a Registration Request (B2B Client)
- **Method:** `POST`
- **Endpoint:** `/client/registerrequest`
- **Headers:** `Content-Type: application/json`
- **JSON Body:**
```json
{
  "companyName": "Acme Printing",
  "contactPerson": "John Doe",
  "email": "johndoe@acme.com",
  "phone": "9812345678",
  "address": "Kathmandu, Nepal",
  "message": "We would like to register for B2B printing services."
}
```

### 1.2 Client Login
- **Method:** `POST`
- **Endpoint:** `/client/login`
- **Headers:** `Content-Type: application/json`
- **JSON Body:**
*(Use the `client_id` (phone) and `password` generated by the Admin approval process)*
```json
{
  "client_id": "9812345678",
  "password": "randompassword" 
}
```
**Important:** Copy the `token` from the response to use in the Authorization headers below.

## 2. PROTECTED CLIENT APIs 
**Required Headings for ALL routes below:**
- `Authorization`: `Bearer <YOUR_CLIENT_TOKEN>`

### 2.1 Get Current User Identity
- **Method:** `GET`
- **Endpoint:** `/client/me`

### 2.2 Get Client Profile Details
- **Method:** `GET`
- **Endpoint:** `/client/profile`

### 2.3 Update Client Profile
- **Method:** `PATCH`
- **Endpoint:** `/client/profile`
- **Headers:** `Content-Type: application/json`
- **JSON Body:**
```json
{
  "phone": "9800001111",
  "address": "Lalitpur, Nepal"
}
```

### 2.4 Client Logout
- **Method:** `POST`
- **Endpoint:** `/client/logout`

## 4. CLIENT DESIGN SUBMISSION APIs
**Required Headings:** `Authorization: Bearer <CLIENT_TOKEN>`

### 4.1 Submit a Custom Design
- **Method:** `POST`
- **Endpoint:** `/client/designs/submit`
- **Headers:** `Content-Type: application/json`
- **JSON Body:**
*(You'll specify the `fileUrl` from your bucket upload. Supabase / S3 bucket URL)*
```json
{
  "fileUrl": "https://your-bucket-url.com/designs/acme_visiting_card.pdf"
}
```

### 4.2 Get Client's Own Design Submissions
- **Method:** `GET`
- **Endpoint:** `/client/designs/my-submissions`

### 4.3 Verify Design Code (For Order Checkout)
- **Method:** `POST`
- **Endpoint:** `/client/designs/verify`
- **Headers:** `Content-Type: application/json`
- **JSON Body:**
```json
{
  "designCode": "DSGN-AABBCCDD"
}
```

### 4.4 Get Approved Design Details by ID
- **Method:** `GET`
- **Endpoint:** `/client/designs/<design_id_here>` *(This expects the DB `id` column, not the `designCode`)*

## 5. PROTECTED ADMIN APIs
**Required Headings for ALL routes below:**
- `Authorization`: `Bearer <YOUR_ADMIN_TOKEN>`

### 5.1 Registration Requests
#### Get All Requests
- **Method:** `GET`
- **Endpoint:** `/admin/registration-requests`

#### Get Request by ID
- **Method:** `GET`
- **Endpoint:** `/admin/registration-requests/<request_id_here>`

#### Approve Request
- **Method:** `POST`
- **Endpoint:** `/admin/registration-requests/<request_id_here>/approve`
*(Returns `client_id` and random `password` which should be given to client)*

#### Reject Request
- **Method:** `PATCH`
- **Endpoint:** `/admin/registration-requests/<request_id_here>/reject`
- **Headers:** `Content-Type: application/json`
- **JSON Body:**
```json
{
  "reason": "Business registration documents are invalid."
}
```

### 5.2 Design Submissions Review Review (Admin)

#### Get Pending Design Submissions
- **Method:** `GET`
- **Endpoint:** `/admin/designs/submissions`

#### Get Design Submission by ID
- **Method:** `GET`
- **Endpoint:** `/admin/designs/submissions/<submission_id_here>`

#### Approve Design Submission
- **Method:** `POST`
- **Endpoint:** `/admin/designs`
- **Headers:** `Content-Type: application/json`
- **JSON Body:**
*(Include the unique DB ID of the pending design submission, and provide the thumbnail preview URL)*
```json
{
  "submissionId": "uuid-of-submission",
  "previewUrl": "https://your-bucket-url.com/designs/preview_acme_card.jpg"
}
```
*(This generates the unique `DSGN-` design code)*

#### Reject Design Submission
- **Method:** `PATCH`
- **Endpoint:** `/admin/designs/submissions/<submission_id_here>/reject`
- **Headers:** `Content-Type: application/json`
- **JSON Body:**
```json
{
  "reason": "The resolution of the design file is too low for printing."
}
```

### 5.3 Approved Designs Management (Admin)

#### Get All Approved Designs
- **Method:** `GET`
- **Endpoint:** `/admin/designs`

#### Get Approved Design By ID
- **Method:** `GET`
- **Endpoint:** `/admin/designs/<design_id_here>`

#### Delete Approved Design
- **Method:** `DELETE`
- **Endpoint:** `/admin/designs/<design_id_here>`
